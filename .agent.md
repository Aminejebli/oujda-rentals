# Oujda Rentals MVP Agent

This agent is tailored for the Oujda Rentals marketplace MVP. It is focused on incremental, realistic improvements to the existing Next.js + React + Tailwind + Supabase app without rewriting the project from scratch.

## Role
- Continue the current Oujda Rentals car rental marketplace MVP.
- Improve credibility, mobile-first UX, local Moroccan trust, and maintain startup-quality polish.
- Add a simple, lightweight admin area separate from the public marketplace.
- Centralize data architecture to reduce hardcoded mocks and duplication.

## Job Scope
- Public marketplace frontend only:
  - Homepage, car listings, agency pages, search, filters, mobile UX, SEO, messaging.
- Admin management area:
  - Simple admin dashboard for managing cars, agencies, availability, featured listings.
  - Keep authentication and admin features lightweight and non-enterprise.
- Data architecture:
  - Cleanly organize cars, agencies, categories, badges, filters, and type definitions.
  - Use centralized data structures in `src/data`, `src/types`, `src/lib`, and `src/services`.
- Image system:
  - Fix all mismatched or unrealistic car images.
  - Standardize aspect ratios, responsive loading, and placeholders.
  - Prefer modern image handling with Next.js `Image` and clean layout.

## Rules
- Preserve the existing project structure and stack when possible.
- Improve progressively; do not rewrite the UI from zero.
- Keep the project beginner-friendly while making it more maintainable.
- Keep everything mobile-first.
- Avoid overengineering backend systems.
- Do not restart or recreate the project.
- Do not build a large enterprise dashboard.

## Preferred architecture
- `src/components` for reusable UI
- `src/sections` for page sections and landing blocks
- `src/data` for centralized mock data
- `src/types` for shared domain types
- `src/lib` for helper clients and domain logic
- `src/hooks` for reusable React hooks
- `src/services` for lightweight data utilities

## Recommended task order
1. Fix wrong car images and standardize image management.
2. Create a public/admin separation with an `/admin` or `/dashboard` route.
3. Centralize car, agency, category, badge, and filter data.
4. Improve responsive, optimized image loading.
5. Refine public homepage, listings, and agency pages for realism and trust.

## When to use this agent
- Use this agent when you want focused work on the Oujda Rentals MVP.
- Pick it over the default agent for tasks that involve public and admin split, data architecture, image quality, and incremental app refinement.
